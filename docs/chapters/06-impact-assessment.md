# Chapter 6: Change Impact Assessment

## Learning Objectives

After completing this chapter, you will be able to:
- Conduct comprehensive change impact assessments
- Identify process, role, and skill impacts
- Create "day in the life" analyses
- Develop impact mitigation strategies

---

## What is Change Impact Assessment?

Change Impact Assessment systematically identifies how a change affects people, processes, technology, and organizational structures.

### Impact Dimensions

| Dimension | Assessment Focus |
|-----------|-----------------|
| **Process** | Workflow changes, new procedures |
| **Role** | Responsibility changes, new roles |
| **Skills** | New competencies required |
| **Tools** | New systems, interfaces |
| **Behavior** | New ways of working |
| **Culture** | Values, norms affected |

---

## Impact Analysis Process

### Step 1: Document Current State
- Map existing processes
- Document current roles
- Identify current tools and systems

### Step 2: Define Future State
- New process workflows
- Changed responsibilities
- New tools and systems

### Step 3: Identify Gaps
- What's changing?
- What's new?
- What's being eliminated?

### Step 4: Assess Impact Severity

| Severity | Definition | OCM Response |
|----------|------------|--------------|
| **High** | Fundamental change to daily work | Intensive support |
| **Medium** | Significant but manageable change | Standard OCM |
| **Low** | Minor adjustments | Light touch |

---

## Key Takeaways

- Impact assessment informs all other OCM activities
- Both organizational and individual impacts must be evaluated
- Impact severity drives OCM intensity and approach
- Day-in-the-life analysis makes impacts tangible
