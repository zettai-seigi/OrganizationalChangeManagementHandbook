## What Each File Contains

Each visual specification includes:

1. **Visual ID** (e.g., 12-Figure-12.1)
2. **Type** (Flowchart, Matrix, Dashboard, etc.)
3. **Title** - Descriptive name
4. **Chapter Reference** - Where it appears
5. **Purpose Description** - Why it's needed
6. **Content Description** - Structure, flow, key elements
7. **Specifications** - Dimensions, colors, fonts, tools
8. **Caption** - For handbook publication
9. **Position in Chapter** - Exact placement
10. **Notes** - Implementation guidance

## Color Palette

All visuals use this consistent color scheme:

- **Primary Blue**: #0066CC - Core practices, governance
- **Accent Orange**: #FF6B35 - Warnings, high priority

- **Warning Yellow**: #FFC107 - Cautions, moderate risk
- **Accent Purple**: #6F42C1 - Excellence, innovation

- **Dark Gray**: #333333 - Text, labels

## How to Use These Specifications

### For Graphic Designers

1. Read the full specification for your assigned visual
2. Use specified dimensions as baseline (can adjust if needed)
3. Maintain color consistency with other handbook visuals
4. Ensure accessibility (colorblind-friendly, high contrast)
5. Create both web (PNG/SVG) and print (PDF) versions
6. Provide editable source files

### For Handbook Authors

1. Reference the Visual ID in chapter text
2. Insert visual at specified position
3. Use the provided caption exactly
4. Verify cross-references are accurate
5. Test clarity at different sizes

### For OCM Practitioners

1. Use these as templates for your organization
2. Customize with your specific data
3. Adapt to your context and needs
4. Share across your change community
5. Provide feedback for improvements
