## Visual ID: 15-Figure-15.3
**Type**: Decision Tree / Framework
**Title**: OCM Decision Rights Framework
**Chapter Reference**: Chapter 15: Governance Framework

### Purpose Description
Clarifies decision-making authority for different types of OCM decisions, showing decision makers, required inputs, and escalation paths to eliminate decision-making bottlenecks.

### Content Description
**Structure**:
- Categorized decision framework with three levels
- Each decision type shows: decision maker, input required, timeframe, escalation path
- Color-coded by decision level (operational, tactical, strategic)
- Visual decision tree for complex decisions

**Visual Flow**:
- Organized by decision authority level
- Left side: Decision categories
- Center: Decision maker and process
- Right side: Required inputs and escalation

**Key Elements**:

**Level 1: Operational Decisions (Day-to-Day)**
Color theme: #28A745 (Success Green)

Decision Categories with Details:

1. **Communication Content Approval**
   - Decision Maker: Project OCM Lead
   - Input Required: Business Sponsor (content), Legal (if needed), Communications Team (brand)
   - Timeframe: 2-3 business days
   - Escalation: To Business Sponsor if controversial content
   - Decision Tree: Simple → Legal review needed? → Yes/No branch
   - Icon: Document with approval stamp

2. **Training Approach Selection**
   - Decision Maker: Project OCM Lead
   - Input Required: Training team, SMEs, impacted users
   - Timeframe: 1 week
   - Escalation: To Project Sponsor if budget impact
   - Icon: Training/education symbol

3. **Change Agent Selection**
   - Decision Maker: Project OCM Lead with Line Managers
   - Input Required: Manager nominations, role requirements
   - Timeframe: 2 weeks
   - Escalation: To HR if performance concerns
   - Icon: Person with network

4. **Measurement Method Selection**
   - Decision Maker: Project OCM Lead
   - Input Required: OCM Practice Lead (methodology), IT (data availability)
   - Timeframe: 1 week
   - Escalation: None typically needed
   - Icon: Dashboard/metrics

**Level 2: Tactical Decisions (Project-Level)**
Color theme: #0066CC (Primary Blue)

Decision Categories with Details:

1. **OCM Approach and Level**
   - Decision Maker: Project OCM Lead
   - Input Required: OCM Practice Lead (standards), Business Sponsor (context), PM (constraints)
   - Timeframe: 1-2 weeks
   - Escalation: To OCM Practice Lead if methodology exception needed
   - Decision Tree: Assessment → Criteria evaluation → Level assignment
   - Icon: Strategic plan

2. **OCM Resource Allocation (Project)**
   - Decision Maker: Project Manager with Business Sponsor
   - Input Required: Project OCM Lead (requirements), PMO (availability)
   - Timeframe: 2-3 weeks
   - Escalation: To IT Leadership if resource contention
   - Icon: Resources/people allocation

3. **Go-Live Readiness**
   - Decision Maker: Business Sponsor
   - Input Required: Project OCM Lead (assessment), PM (technical readiness), Business SMEs (operational readiness)
   - Timeframe: Determined by project plan
   - Escalation: To Executive Sponsor if "Not Ready" determination
   - Decision Tree: Complex multi-criteria assessment → Criteria met? → Go/No-Go/Conditional
   - Icon: Traffic light or go-live symbol

4. **Resistance Escalation**
   - Decision Maker: Business Sponsor
   - Input Required: Project OCM Lead (assessment), HR (policy), Line Manager (context)
   - Timeframe: 1 week
   - Escalation: To HR Leadership if performance action needed
   - Decision Tree: Resistance identified → Root cause → Engagement tried? → Escalation decision
   - Icon: Alert with resolution path

**Level 3: Strategic Decisions (Enterprise-Level)**
Color theme: #FF6B35 (Accent Orange)

Decision Categories with Details:

1. **OCM Methodology Standards**
   - Decision Maker: OCM Practice Lead
   - Input Required: CoE members, CoP feedback, external benchmarks
   - Timeframe: 3-6 months for major updates
   - Escalation: To OCM Steering Committee for significant changes
   - Icon: Standards document

2. **OCM Policy Approval**
   - Decision Maker: OCM Steering Committee
   - Input Required: OCM Practice Lead (proposal), Legal (compliance), affected stakeholders
   - Timeframe: 1-3 months
   - Escalation: To Executive Leadership if cross-functional impact
   - Icon: Policy/governance

3. **OCM Resource Allocation (Enterprise)**
   - Decision Maker: OCM Steering Committee
   - Input Required: OCM Practice Lead (needs), IT Leadership (priorities), Finance (budget)
   - Timeframe: Aligned with budget cycles
   - Escalation: To Executive Leadership if significant investment
   - Icon: Budget/investment

4. **OCM Policy Exceptions**
   - Decision Maker: OCM Practice Lead
   - Input Required: Project OCM Lead (justification), OCM Steering (if major exception)
   - Timeframe: 1-2 weeks
   - Escalation: To Steering Committee if policy precedent would be set
   - Decision Tree: Exception requested → Risk assessment → Impact analysis → Approve/Deny/Modify
   - Icon: Exception/variance symbol

**Escalation Path Visualization** (Right side):
- Vertical flow showing escalation levels
- Arrows indicating when to escalate
- Timeframes for escalation response
- Contact points at each level

  - Level 1 background: Light green tint #E6F9E6

  - Decision tree text: 10pt Regular

## Visual ID: 15-Figure-15.4
**Type**: Process Flow with Criteria
**Title**: OCM Requirements Determination Process
**Chapter Reference**: Chapter 15: Governance Framework

### Purpose Description
Illustrates the systematic process for determining OCM requirements for initiatives, including assessment criteria, level determination, and resource planning to ensure appropriate OCM support.

### Content Description
**Structure**:
- Horizontal process flow with 5 phases
- Integrated criteria table for level determination
- Decision diamonds for key gates
- Resource allocation matrix
- Feedback loop for continuous improvement

**Visual Flow**:
- Left to right: Initiative intake through ongoing oversight
- Decision gates determine OCM level
- Resource planning follows level determination

**Key Elements**:

**Phase 1: Initiative Assessment**
- Input: New initiative or project proposal
- Activities:
  - Gather initiative details
  - Identify impacted stakeholders
  - Assess change characteristics
  - Complete assessment questionnaire
- Deliverable: "Initiative Profile"
- Owner: Project Manager with OCM support
- Timeframe: 1-2 weeks
- Icon: Clipboard with assessment form

**Phase 2: OCM Level Determination**
- Input: Initiative Profile
- Decision diamond: "Evaluate against criteria"

**OCM Level Criteria Table** (embedded in this phase):

| Factor | Level 1 (Light) | Level 2 (Standard) | Level 3 (Intensive) | Weight |
|--------|----------------|--------------------|--------------------|--------|
| **Impacted Users** | <100 | 100-1,000 | >1,000 | 20% |
| **Impact Severity** | Low - Minor changes | Medium - Significant changes | High - Fundamental changes | 25% |
| **Stakeholder Complexity** | Single team/dept | Multiple departments | Cross-functional/enterprise | 15% |
| **Change History** | Positive - Change-ready | Mixed - Some resistance | Negative - Change fatigue | 20% |
| **Strategic Importance** | Low - Operational | Medium - Departmental goals | High - Strategic objectives | 20% |

Scoring:
- Calculate weighted score
- Score 0-30: Level 1 (Light)
- Score 31-65: Level 2 (Standard)
- Score 66-100: Level 3 (Intensive)

Decision outcomes (3 branches):
- Level 1: Light OCM
- Level 2: Standard OCM
- Level 3: Intensive OCM

**Phase 3: Resource Planning**
Three parallel paths based on level:

**Level 1 Resources** (Green path):
- OCM Role: PM with OCM coaching (10-15% time)
- Support: Templates and self-service tools
- Budget: 1-2% of project budget
- Deliverables: Basic communication and training
- Icon: Single person with toolkit

**Level 2 Resources** (Blue path):
- OCM Role: Dedicated OCM Lead (50-75% time)
- Support: CoE consultation, standard tools
- Budget: 3-5% of project budget
- Deliverables: Full OCM plan and execution
- Icon: Small team

**Level 3 Resources** (Orange path):
- OCM Role: OCM Lead + team (multiple full-time)
- Support: Executive sponsor, extended team, specialized tools
- Budget: 5-10% of project budget
- Deliverables: Comprehensive OCM program
- Icon: Large team with support structure

**Phase 4: Plan Review & Approval**
- Activity: Review OCM approach and plan
- Reviewers:
  - OCM Practice Lead (quality review)
  - Business Sponsor (approval)
  - PMO (alignment check)
- Decision diamond: "Plan adequate and approved?"
  - If No: Return to Phase 3 for revision
  - If Yes: Proceed to Phase 5
- Deliverable: "Approved OCM Plan"
- Timeframe: 1-2 weeks
- Icon: Document with approval checkmarks

**Phase 5: Ongoing Oversight**
- Activities:
  - Monitor OCM execution
  - Track metrics and milestones
  - Review progress regularly
  - Adjust as needed
- Governance forums:
  - Weekly: Project team
  - Bi-weekly: Steering committee
  - Monthly: OCM governance review
- Deliverable: "OCM Status Reports"
- Owner: Project OCM Lead
- Icon: Dashboard with monitoring

**Feedback Loop**:
- From Phase 5 back to Phase 2
- Dashed purple line
- Label: "Level adjustment if circumstances change"
- Trigger: Scope change, stakeholder change, timeline change

**Supporting Elements**:
- Risk indicators: Red flags for common issues at each phase
- Shortcuts: Dashed gray lines showing expedited paths for urgent initiatives
- Templates: Reference to standard templates at each phase

### Specifications
- **Dimensions**: 1800px × 1000px (wide landscape)
- **Phase Boxes**: 320px × 700px each
- **Criteria Table**: 900px × 300px (embedded in Phase 2)
- **Resource Boxes**: 300px × 180px each
- **Colors**:
  - Phase boxes: #F8F9FA (Light Gray) with colored top border
  - Level 1 path: #28A745 (Success Green)
  - Level 2 path: #0066CC (Primary Blue)
  - Level 3 path: #FF6B35 (Accent Orange)
  - Decision diamonds: #FFC107 (Warning Yellow)
  - Feedback loop: #6F42C1 (Accent Purple) dashed
  - Criteria table: Standard table format with alternating row colors
- **Fonts**:
  - Phase titles: 16pt Bold
  - Activities: 11pt Regular
  - Criteria table: 10pt Regular
  - Resource details: 11pt Regular
  - Timeframes: 10pt Italic
- **Tools**: Lucidchart, Visio, or Process mapping software

### Caption
**Figure 15.4: OCM Requirements Determination Process** - Systematic assessment determines appropriate OCM level and resources for each initiative. Clear criteria ensure consistent, objective decisions about OCM investment.

### Position in Chapter
Insert in "Governance Processes" section, specifically in "OCM Requirements Determination" subsection.

### Notes
- Make criteria table easily customizable for organizations
- Show that level determination is objective, based on measurable factors
- Include examples for each level to illustrate differences
- Consider creating Excel calculator tool to accompany visual
- Add note that levels can be adjusted if circumstances change significantly
- Emphasize that higher levels are not "better" - appropriateness is key
